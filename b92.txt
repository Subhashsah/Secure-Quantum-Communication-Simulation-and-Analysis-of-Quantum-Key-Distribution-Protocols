 The B92 protocol is a quantum key distribution (QKD) protocol proposed in 1992 by Charles H. Bennett.
 It is a simplified version of BB84, using only two non-orthogonal quantum states instead of four.
 Unlike BB84 (which uses 4 states), B92 uses only 2 non-orthogonal states

 “Two quantum states are non-orthogonal if their inner product is non-zero, meaning they cannot be perfectly distinguished by any measurement. In B92, non-orthogonal states provide security against eavesdropping.”
 
In quantum mechanics, two states are orthogonal if they can be perfectly distinguished by a measurement.

Orthogonal states → inner product = 0
Non-orthogonal states → inner product ≠ 0

Alice uses
• Bit 0 → |H⟩ (0°)
• Bit 1 → |+⟩ (45°)

Bob measures using
• |V⟩ (orthogonal to |H⟩)
• |−⟩ (orthogonal to |+⟩)

Goal:
Securely share a secret key between two parties using quantum mechanics, such that any eavesdropping can be detected.

Core Idea:
In quantum mechanics, non-orthogonal states cannot be perfectly distinguished
If an eavesdropper (Eve) tries to measure them, she inevitably introduces errors
These errors alert the legitimate users

Detector Clicks:
A detector click means that Bob’s measuring device has detected a photon in the chosen measurement state.

5. POVM (Positive Operator-Valued Measure)
A POVM is a measurement that allows three kinds of outcomes instead of just two.
A POVM allows Bob to say:
✅ “I know it is bit 0”
✅ “I know it is bit 1”
❌ “I don’t know”

Bob’s measurement is not projective, but a POVM:
Can give:
Conclusive result:
A conclusive result means Bob gets a detector click and can confidently infer the bit value.

Inconclusive result:
An inconclusive result means Bob’s detector does not click, so Bob cannot determine the bit.

Alice generates a random classical bit string and encodes each bit into a quantum state using two non-orthogonal states. These quantum states are transmitted to Bob through a quantum channel. Bob performs measurements using states orthogonal to Alice’s signal states. When Bob obtains a conclusive measurement outcome, a shared key bit is generated; inconclusive measurement outcomes are discarded.


Working of B92 protocol
Step 1: Alice Prepares Qubits
 - Alice generates a random classical bit string
 - Encodes each bit into a quantum state using two non-orthogonal states

Step 2: Send to the Bob
 - These quantum states are transmitted to Bob through a quantum channel

Step 3: Bob Measures the Qubits
 - Bob performs measurements using states orthogonal to Alice’s signal states

Step 4: Bob Announces Detection Positions
 - Bob announces only the time slots where he got a conclusive result
 - He does NOT reveal measurement outcomes

Step 5: Alice Confirms Bits
 - Alice knows which bit she sent at those positions
 - These bits form the raw key

Step 6: Eavesdropping Detection
 - Alice and Bob publicly compare a subset of the raw key
 - If error rate > threshold → Eve detected → abort
